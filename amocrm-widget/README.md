# Виджет "Отправка сделки в таблицу"

Добавляет в карточку сделки amoCRM кнопку, по клику на которую данные сделки
отправляются в Google Таблицу через роут [`/api/send-lead`](../app/api/send-lead/route.ts)
этого модуля.

## Перед сборкой

1. Задеплойте модуль (Next.js-приложение) и узнайте его публичный домен.
2. В [`script.js`](script.js) замените значение `ENDPOINT_URL` на реальный адрес:
   `https://<ваш-домен>/api/send-lead`.
3. Добавьте файл `images/logo.png` (128×128) — amoCRM требует лого для загрузки пакета виджета.

## Сборка пакета

amoCRM ожидает zip-архив с содержимым **корня** этой папки (без вложенной директории):

```
manifest.json
script.js
style.css
i18n/
  ru.json
  en.json
images/
  logo.png
```

## Установка в amoCRM

1. Настройки → Интеграции → Создать интеграцию.
2. Загрузите собранный zip-архив как виджет.
3. Включите виджет в списке установленных.
4. Откройте любую сделку — в правой панели появится блок с кнопкой
   «Отправить в таблицу». По клику сделка будет отправлена на `/api/send-lead`.

## Как это работает

- ID сделки виджет берёт из URL страницы (`/leads/detail/<id>`), поэтому не
  зависит от внутренних деталей SDK amoCRM.
- Роут `/api/send-lead` вызывается напрямую с домена amoCRM (`*.amocrm.ru`),
  поэтому на сервере настроен CORS именно под этот паттерн домена
  (см. `corsHeaders` в [`route.ts`](../app/api/send-lead/route.ts)).
- Вебхук по пути `/api` (form-data) продолжает работать как раньше —
  обе точки входа используют общую функцию `syncLeadToSheet`
  из [`lib/syncLead.ts`](../lib/syncLead.ts).
